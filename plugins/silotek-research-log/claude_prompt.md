# Claude에게 연구일지 작성 요청하기

연구/개발 작업이 끝난 뒤 Claude에게 대화 전체를 정리해달라고 할 때 사용하는 프롬프트입니다.

---

## 사용 방법

1. Claude와의 대화 끝에 아래 프롬프트를 **그대로 복사해서 붙여넣기**
2. Claude가 YAML 형식으로 연구일지 초안 출력
3. 초안을 `inputs/YYYY-MM-DD-주제.yaml` 파일로 저장
4. `node build.js inputs/파일명.yaml` 실행 → DOCX 생성

---

## 표준 프롬프트 (복사해서 사용)

```
지금까지의 대화 내용을 사일로텍 연구일지 YAML 형식으로 정리해줘.

## 출력 요구사항

1. 출력 형식은 YAML만 (다른 설명 없이 코드 블록 안에 YAML만)
2. 인코딩: UTF-8
3. 파일명 제안: YYYY-MM-DD-주제.yaml

## 포함해야 할 내용

### 필수
- title: "연구 일지"
- subtitle: 연구 주제 한 줄
- meta:
  - 연구 주제: 상세 설명
  - 작성일: 오늘 날짜 (YYYY년 M월 D일)
  - 연구 단계: (기술 검증 / 구현 / 운영 / 분석 중 택)
  - 분류: 관련 기술 키워드 나열

### 본문 섹션 (sections)
다음 장 구성을 따라:
1. 연구 배경 및 목적
2. 방법 / 시스템 구성 / 실험 설계
3. 실험 과정 및 주요 발견
4. 핵심 발견 및 한계
5. 종합 분석 (선택)
6. 결론 및 향후 과제
7. 참고 사항 (선택)

## 익명화 원칙 (중요)

다음은 반드시 익명화 또는 일반화:
- 회사명 (사일로텍은 유지, 고객사명은 제외)
- 실제 사람 이름
- 실제 프로젝트명
- 구체적 제품명
- 실제 데이터베이스 이름, URL, API 엔드포인트
- 실제 쿼리문에 포함된 회사 특화 용어

일반화 예시:
- "명화공업 EOP 시험" → "자동차 부품 내구 시험"
- "TLMS" → "시험 데이터 분석 시스템"
- "silotek_hub 컬렉션" → "프로젝트 벡터 DB 컬렉션"
- "FW Update 관련 문서" → "특정 키워드 기반 검색"

## 이미지 참조

이미지는 figures/ 폴더에 있다고 가정하고 참조:
- 예: path: "../figures/diag_a.png"

캡션은 "[그림1] 설명" 형식.

어떤 이미지가 있는지 내가 목록을 알려주면 그것에 맞춰 참조해.
목록을 알려주지 않으면 "[그림 자리]" 주석으로 표시만 해두고 실제 path는 비워둬.

## 서식 규칙

- 중요한 키워드는 **굵게** 표시 (마크다운 스타일)
- 코드 블록은 code: 요소로
- 테이블은 table: 요소로 (headers + rows)
- 중요 주의사항은 note: 요소로

## 스타일

- 존댓말 아닌 서술체 (“~이다”, “~하였다”)
- 간결하고 명료하게
- 기술적 정확성 유지
- 엔지니어 독자를 염두에 둔 표현

## 예시 구조 (참고만)

```yaml
title: "연구 일지"
subtitle: "여기에 부제"
meta:
  연구 주제: "..."
  작성일: "2026년 4월 19일"
  연구 단계: "기술 검증"
  분류: "AI/ML, RAG"
sections:
  - h1: "1. 배경 및 목적"
  - h2: "1.1 배경"
  - p: "서술 내용..."
  - h2: "1.2 목적"
  - bullets:
      - "목적 1"
      - "목적 2"
  - h1: "2. 방법"
  - image:
      path: "../figures/diag_a.png"
      caption: "[그림1] 아키텍처"
  - table:
      headers: ["항목", "값"]
      rows:
        - ["A", "1"]
```

이제 이번 대화 내용을 이 형식으로 정리해서 YAML 하나 만들어줘.
```

## 고급 사용법

### 특정 섹션만 재생성

이미 작성한 YAML의 특정 장만 Claude에게 다시 써달라고 할 수 있음:

```
아래 YAML에서 '4장 핵심 발견' 섹션만 더 상세하게 다시 써줘. 
다른 섹션은 그대로 두고.

[YAML 붙여넣기]
```

### 영문 연구일지

```
위 프롬프트와 동일하되, 본문을 영어로 써줘. 메타데이터는 한글 유지.
```

### 요약 버전

```
위 프롬프트와 동일하되, 섹션을 3개로 압축한 요약 버전으로 작성.
(배경, 주요 발견, 결론만)
```
